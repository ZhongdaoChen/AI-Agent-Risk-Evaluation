🤖 Agent能力/爆炸半径分析
🤖 Agent能力/爆炸半径分析（权重 25%）
枚举这个 AI Agent 能做什么（爆炸半径），而非找代码漏洞。由 LLM 语义分析。
  • 终端/Shell 执行：−25分（最高风险）
  • 代码解释器/REPL：−22分
  • 文件系统写入/删除：−18分
  • 数据库写入：−18分
  • 云服务/邮件/外部 API 等
高分(75+)=低影响能力；低分(<35)=几乎可完全控制宿主系统
 🔧 Skill安全质量
🔧 Skill 安全质量（权重 12%）
聚焦 Skill 本身的恶意风险，而非普通代码质量缺陷。
  • 恶意外传 / 凭证收集
  • 混淆执行 / 远程脚本
  • 后门 / 持久化 / 恶意链路
  • Medium / Low 默认不计分
只保留主观恶意的 High / Critical，并按文件聚合展示
 🛡️ Agent安全护栏
🛡️ Agent安全护栏（权重 20%）
评估是否有有效的安全控制机制防止 Agent 失控或被滥用。基准 40 分（保守）。
  • 人工审批/确认循环：+15分
  • 步骤/轮次上限：+15分
  • 工具白名单/权限系统：+10分
  • Prompt Injection 防御：+8分
  • exec/eval 执行 LLM 输出：−25分
用户输入直接注入 System Prompt −15分；完全无安全机制 −15分
 ⭐ 开源项目声誉与活跃度
⭐ 开源项目声誉与活跃度（权重 15%）
通过 GitHub 公开数据判断项目的社区信任度、维护状态和安全成熟度。
  • Stars 数量：社区验证程度
  • 最后提交时间：维护活跃度
  • 贡献者数量：Bus Factor 风险
  • SECURITY.md：漏洞响应机制
  • CI/CD 工作流：代码变更是否自动检验
⚠️ 已归档仓库 −30分；停止维护超1年 +0分
 🌐 deps.dev
🌐 依赖包健康度（权重 5%）
通过 deps.dev API 评估项目本身作为包的健康状况，交叉验证依赖许可证和漏洞。
  • deps.dev 项目记录：透明度验证
  • Open Issues 积压：未处理问题风险
  • 依赖者数量：供应链爆炸半径
  • 安全公告记录：每条 −10 分
依赖者越多，该包越是供应链攻击的高价值目标
 📦 依赖漏洞
📦 依赖漏洞扫描（权重 15%）
扫描所有直接依赖的已知 CVE，通过 OSV.dev 数据库查询。
  • CRITICAL（CVSS≥9.0）：每条 −25分
  • HIGH（CVSS 7.0-8.9）：每条 −12分
  • MEDIUM（CVSS 4.0-6.9）：每条 −5分
  • 支持 Python / Node.js / Go / Rust
有 CRITICAL 漏洞时强烈建议暂停使用直至修复
 🔒 隐私
🔒 数据隐私（权重 10%）
评估是否存在用户数据泄露风险，包括对话内容被记录、PII 处理不当等。
  • PRIVACY.md 文档：+15分
  • 数据匿名化/脱敏代码：+10分
  • 遥测默认关闭：+10分
  • 将用户输入写入日志：−20分
  • 第三方行为追踪 SDK：−10分
遥测默认开启（无用户同意）：−15分
 ⛓️ 供应链
⛓️ 供应链完整性（权重 5%）
评估依赖管理和分发机制是否能防止供应链攻击（依赖替换、模型篡改）。
  • 依赖锁文件（poetry.lock 等）：+20分
  • 无锁文件：−15分（版本漂移风险）
  • SBOM 软件物料清单：+10分
  • 模型文件哈希校验：+5分
  • 下载模型但无哈希校验：−10分
无锁文件时，攻击者可通过发布恶意新版本实施供应链攻击
 🐳 运行时
🐳 运行时隔离（权重 5%）
评估 Agent 是否被容器化或沙箱化部署，防止逃逸到宿主环境。
  • Dockerfile + 非 root 用户：+35分
  • 以 root 运行容器：−20分
  • 挂载 Docker socket：−20分（等同宿主 root）
  • --privileged 标志：−20分（极度危险）
  • 沙箱平台集成（e2b/modal等）：+10分
完全无隔离配置：−10分（Agent 直接在宿主机裸跑）
  • •System Prompt 从远程 URL 动态加载的内容
  • •工具调用链在运行时动态组合的行为
  • •MCP Server 连接后实际获得的权限
  • •Agent 运行时的真实行为与副作用
  • •由于文件摘要和提取策略，分析超大型项目时覆盖率会降低（如 OpenClaw）
结果供参考，不构成最终安全结论，请联系 Security 进行进一步审核，感谢！
